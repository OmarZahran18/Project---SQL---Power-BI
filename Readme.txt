# Supermarket Sales Analysis  

## 📌 Project Overview  
This project analyzes supermarket sales data to uncover valuable insights about customer behavior, sales performance, and operational trends. The analysis was conducted using **SQL**, and the results were visualized through an interactive **Power BI** dashboard.  

The aim of this project is to demonstrate data analytics skills by applying them to a real-world dataset and producing actionable insights.  

---

## 🗂️ Dataset  
- **Source:** Supermarket Sales Dataset  
- **Description:** This dataset contains sales records from a supermarket, including customer information, product lines, and transactional details.  
- **Columns Included:**
  - `Branch`, `City`, `Customer Type`, `Gender`
  - `Product Line`, `Unit Price`, `Quantity`
  - `Tax 5%`, `Total`, `Date`, `Time`
  - `Payment`, `Cost of Goods Sold (COGS)`
  - `Gross Margin Percentage`, `Gross Income`, `Rating`

---

## 🔑 Key Insights  
- **Branch Performance:** Identified the branch with the highest total sales and average ratings.  
- **Product Trends:** Discovered which product lines generated the highest sales and ratings.  
- **Customer Analysis:** Compared spending patterns between genders and customer types.  
- **Peak Times:** Analyzed sales trends during different times of the day.  
- **Payment Preferences:** Determined the most commonly used payment methods.  
- **Regional Performance:** Evaluated city-wise total sales and taxes collected.  

---

## 🛠️ Tools Used  
### **SQL**  
- Cleaned and explored the dataset.  
- Wrote queries to extract key metrics like total sales, average ratings, and payment distributions.  
- Used conditional logic to identify peak hours of sales.  

### **Power BI**  
- Built an interactive dashboard with slicers, filters, and drill-down capabilities.  
- Visualized sales trends, branch performance, and customer preferences.  
- Designed charts such as bar charts, pie charts, and line graphs to present insights clearly.
